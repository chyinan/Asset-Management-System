# UI 审计概览（2025-11-19）

> 范围：`src/views`、`layouts/AppLayout.vue`、`store/auth.ts`、Element Plus 默认样式。目的：摸底当前 UI 资产，定位视觉/交互/信息架构薄弱点，为主题与组件升级提供依据。

## 页面级检查

| 页面 | 入口/文件 | 主要模块 | 视觉/布局现状 | 交互与数据痛点 |
| --- | --- | --- | --- | --- |
| 登录 | `views/auth/Login.vue` | 标题、账号密码表单、CTA 按钮 | 纯铺满渐变背景，卡片宽度固定 360px，层次不足 | 无品牌元素、无表单反馈色，缺辅助说明/安全提示 |
| 仪表盘 | `views/dashboard/Dashboard.vue` | KPI 卡片、快速操作区 | 栅格为 4x 等宽卡片，色彩单一，缺少图标/趋势信息 | KPI 数据为写死数字，操作按钮无分组或状态提示 |

## 组件 & 状态清单

- **Layout**：`AppLayout.vue` 使用 Element Plus Container，未抽象导航项配置，主题色与 Element 默认灰底不一致；缺少折叠/紧凑模式。
- **表单**：广泛使用 `el-form` + `label-width`，但字段间距随组件内联样式变化，验证提示依赖默认红色，缺少描述文本。
- **表格**：所有数据页均依赖 `el-table`，未启用固定表头、可滚动区域、空状态插槽；列宽和操作区未对齐，Tag spacing 依靠自定义类。
- **反馈**：Loading 状态多使用 `v-loading` 或全局 `saving`，缺 Skeleton、空数据说明、Toast 规范，按钮 loading 状态使用较少。
- **图表**：目前无图表组件；仪表盘为纯文本 KPI。
- **状态管理**：`store/auth.ts` 提供 `permissions`，页面常用 `usePermission` 来控制按钮显隐，但 UI 未对无权限场景提供视觉占位或提示。

## 风格与主题现状

- 色彩：主体灰白 (`#f6f8fa`) + 深色侧栏 (`#101828`) + Element Plus 默认主色 (`#409EFF`)，未形成系统；渐变只在登录页使用且与主色无关。
- 字体：全局 `Inter`，中文 fallback 为系统默认；字号、间距、圆角、阴影缺乏 token，导致同类组件风格各异。
- Icon：使用 Element Plus Icon，缺少尺寸及边距约束。

## 建议摘要（支撑后续主题配置）

1. **主题 Token 化**：抽象色板（基础、中性色、语义色、渐变）、排版、Spacing、Shadow、Radius，通过 CSS 变量 + TS 导出，向后续组件开放。
2. **语义分层**：登录/仪表盘使用柔和蓝灰渐变背景与玻璃拟态卡片，表格场景保持高对比中性底色；定义 KPI、操作、警示的专属色阶。
3. **组件规范表**：建立按钮、表单、表格、卡片的状态图谱（默认/hover/active/disabled/loading），并在主题中给出 Token。
4. **数据可视化占位**：在仪表盘/库存页预留图表容器及卡片布局规范，统一卡片高度、padding、media query 行为。
5. **可访问性**：对比度 ≥ 4.5:1，键盘焦点样式纳入主题，常用交互提供 `aria-label`/描述文本。

> 本审计文档后续可作为设计规范与 Storybook 的输入，确保布局、视觉、交互和数据层面的优化可量化跟踪。
